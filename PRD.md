# 智慧打印机管理平台 — 项目需求文档 (PRD)

## 1. 项目概述

### 1.1 产品定位
面向企业与个人用户的智慧打印机管理平台，提供设备实时监控、云端 AI 故障诊断、文件打印、驱动下载及电子文档管理一站式服务。

### 1.2 目标用户
| 角色 | 场景 |
|------|------|
| 企业 IT 管理员 | 批量管理办公室打印机设备，监控状态，远程诊断故障 |
| 普通员工/个人用户 | 提交打印任务，下载驱动，获取故障自助排查指导 |
| 运维支持人员 | 接收告警，远程诊断，推送维修方案 |

### 1.3 核心价值
- **降本**：AI 自动诊断减少 70% 人工运维成本
- **提效**：实时监控 + 告警，故障响应时间从小时级降至分钟级
- **一站式**：打印、驱动、诊断、文档管理，无需多系统切换

## 2. 功能模块

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 设备列表仪表盘 | 展示所有打印机设备的卡片/表格视图，含设备名称、型号、IP、位置 | P0 |
| 实时状态展示 | 在线/离线/繁忙/错误状态，墨量/碳粉余量，纸张余量 | P0 |
| 连接状况监控 | SNMP / IPP / WebSocket 实时连接状态，网络延迟 | P0 |
| 打印队列查看 | 当前排队任务数，预计等待时间 | P1 |
| 历史运行统计 | 打印页数统计（日/周/月），故障频次，正常运行时间 | P2 |
| 异常告警推送 | 缺纸/卡纸/墨尽/离线，支持邮件/短信/Webhook | P1 |

### 2.2 云端 AI 故障诊断
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 智能故障识别 | 基于设备错误码 + 日志 + 传感器数据，AI 自动判定故障类型 | P0 |
| 自然语言问答 | 用户描述故障现象，AI 给出诊断结论与解决方案 | P0 |
| 根因分析 | 多设备关联分析，定位是单设备故障还是网络/服务器侧问题 | P1 |

### 2.3 文件打印服务

#### 2.3.1 打印文件上传

> **实现要求：非静态占位，需前后端完整打通**

| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 拖拽/点击上传 | 支持拖拽文件至上传区域或点击选择，支持批量上传 | P0 |
| 多格式支持 | PDF / Word (.doc/.docx) / Excel (.xls/.xlsx) / PPT (.ppt/.pptx) / 图片 (.png/.jpg/.bmp) / 文本 (.txt) | P0 |
| 上传进度 | 实时显示单个文件及总体的上传进度条 | P0 |
| 文件大小限制 | 单文件 ≤ 100MB，单次批量 ≤ 500MB，超限友好提示 | P1 |
| 文件预览 | 上传后点击缩略图可预览文件内容（前 3 页） | P1 |
| 云端文件拉取 | 支持从 OneDrive / Google Drive / 企业网盘直接选择文件 | P2 |

**落地实现要点：**
```
前端                                   后端
───                                   ───
拖拽区域 (react-dropzone)      →     POST /api/files/upload
分片上传 (SparkMD5 + slice)    →     分片合并 + 完整性校验
上传进度条 (axios onUploadProgress)   文件类型魔数校验（防伪造后缀）
上传完成回调 (文件ID + 元数据)  →     写入 file_records 表 + MinIO 存储
```


### 3.4 设备通信协议

| 协议 | 用途 |
|------|------|
| **SNMP** | 读取打印机状态（墨量、页数、错误码）、设备发现 |
| **IPP** | 提交打印任务、查询打印队列、获取作业状态 |
| **mDNS / WS-Discovery** | 局域网自动发现打印机设备 |
| **WebSocket** | 设备事件实时推送（需打印机支持或通过 Agent 中转） |
| **MQTT** | 跨网络设备状态上报（适用于大量设备场景） |

## 5. 非功能性需求

| 维度 | 要求 |
|------|------|
| **性能** | 设备状态刷新延迟 ≤ 3s；AI 诊断响应 ≤ 10s；页面首屏 ≤ 2s |
| **可用性** | 核心服务 99.9% SLA；支持水平扩展 |
| **安全** | JWT 认证 + RBAC 权限；文件传输 TLS 加密；打印任务日志审计 |
| **兼容性** | 首期深度适配 HP Laser MFP 136a；逐步覆盖主流品牌：HP, Canon, Epson, Brother, Kyocera, Ricoh, Xerox 等 |
| **可扩展** | 支持 OAuth2.0 / LDAP 对接企业账号体系；支持自定义打印规则引擎 |

## 7. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|----------|
| 打印机品牌协议差异大 | SNMP OID 不统一 | 建立品牌-型号 MIB 映射库，逐步覆盖 |
| AI 诊断准确率不足 | 用户信任度下降 | 规则引擎兜底 + 人工审核机制 + 持续微调 |
| 局域网打印机发现困难 | 设备无法自动注册 | 支持手动添加 IP + Agent 客户端辅助发现 |
| 大文件打印传输慢 | 用户体验差 | 文件分片上传 + CDN 加速 + 本地缓存 |

- 支持设备品牌数：≥ 5 家
- AI 诊断准确率：≥ 85%
- 设备状态采集间隔：≤ 5 秒
- 驱动库覆盖型号数：≥ 500 个
- 单页打印提交流程步骤：≤ 4 步
